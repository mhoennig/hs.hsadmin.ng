# PR#226: Improve test-code-coverage to fix failing JaCoCo check

## The Problem

The JaCoCo test coverage verification failed because several recently added or changed production classes were not properly covered by unit tests.
This made otherwise unrelated branches fail the build quality gate.


## The Cause

We currently have no CI/CD pipeline and missed to run all checks for previous PRs.

## The Solution

### Improve Test-Code-Coverage

Some classes and methods had bad test-code-coverage, but were unused, this got removed:

- class `AuthenticatedHttpServletRequestWrapper`
- method `HsAccountEntity.validatePasswordHash`.

### Local CI helper script `git-watch-origin-and-test`

The helper script `git-pull-and-if-origin-changed-run-tests` was renamed to `git-watch-origin-and-test`.

It's now easier to run all tests and checks regularly, especially if you have a spare maschine to run the tests.

The helper script can now be installed via `--bin`; it also refuses to run from its own repository checkout.

It also  supports `--run` for an initial build of the current branch, even without new commits,
and `--new` limits to recently active remote branches, to avoid old branches to be built.

As soon as we have the capacity, we should set up a CI/CD environment again instead of relying on local helper scripts.

## Additional Changes

`LdapArgon2Hash` was marked as a Lombok utility class because it only exposes static behavior.

Spotless formatting was applied to some files;
this includes asterisk ('*') imports to be replaced by single imports.

Most of the tests were generated by agentic AI, the used Ai Agent Config was added to the repo.

## Verification

With the changes in this PR, `gw jacocoTestCoverageVerification` passes again after all tests had run.

This can be tested by running `gw-test --all --rerun` locally.
